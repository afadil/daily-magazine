Resolute Raccoon lands.

Ubuntu 26.04 LTS ships today on Linux 7.0, with Rust promoted from "experiment" to first-class kernel citizen and ten years of paid support behind it.

Canonical's third-of-the-decade LTS arrives with GNOME 50 on a Wayland-only session, memory-safe Rust core utilities, systemd 259 with mandatory cgroup v2, Dracut as the default initramfs generator, APT 3 (with apt-key finally gone), and TPM-backed encryption with post-quantum defaults. Direct in-place upgrades from 24.04 LTS won't open until point release 26.04.1 in August — patient updaters win again.

Managed Agents opens public beta — and a CLI you'll actually keep aliased.

Anthropic's managed harness, in beta since April 8, removes the boring half of building production agents: sandboxing, session state, credential handling, persistence, and SSE streaming all happen on Anthropic's side. You define behavior, tools, constraints; the platform runs it. Notion, Rakuten and Sentry are already shipping on it.

The new ant CLI is the under-discussed half: a real command-line client for the Claude API with native Claude Code integration and YAML-versioned API resources. Sessions cost $0.08/hr on top of token pricing; access is open via the managed-agents-2026-04-01 header.

Electrons in graphene flow like a frictionless liquid — and break a 173-year-old law.

Researchers at the IISc, in collaboration with the National Institute for Materials Science in Japan, reported deviations from the Wiedemann–Franz law of more than 200 times at low temperatures. The rule, which has tied heat and electrical conduction together since 1853, simply no longer holds for the Dirac fluid they observed inside ultraclean graphene.

What the team measured behaves less like a metal and more like a hydrodynamic plasma: minimal viscosity, conductance set by a universal quantum constant, and momentum that propagates without scattering off lattice defects.

For builders of quantum sensors, this is not a curiosity. A material whose carriers behave as a near-perfect fluid at low temperature is exactly the kind of substrate you want for ultra-sensitive thermometry, current sensing, and the long line of mid-2020s graphene devices waiting for a cleaner electron bath.

Anthropic crosses $30B ARR — and passes OpenAI for the first time.

Anthropic disclosed earlier this month that its annualized revenue run rate has crossed thirty billion dollars, up from roughly nine billion at the close of 2025. The figure puts it above OpenAI's reported ~$25B for the first time in either company's history.

The shape of the growth is what matters more than the trophy: enterprise spend on agents and managed runtimes — not consumer subscriptions — drives the curve. Claude Code, the API, and the Managed Agents harness are the three line items doing the lifting.

Pricing has stayed flat through the move. Opus 4.7 launched on April 16 at the same price as Opus 4.6, and the brief test that pulled Claude Code from the $20 Pro plan was reversed within hours after public uproar.

The signal for architects: Anthropic is funding multi-year roadmaps from operating revenue, not just venture rounds. The procurement story is now plausible at every scale.

Google's leadership confirmed that 75% of new code created inside the company is now generated by AI and reviewed by human engineers, up from 50% last fall. For an architect, the implication is no longer "will my team adopt this." It's "what does code review look like when humans review more than they author."

Direct hit · You drive Claude Code daily

"Over-editing" — the failure mode no benchmark catches.

A new note circulating the front page coins the term: when a model edits more code than the task requires — moving an import here, "fixing" an unrelated whitespace there, refactoring the surrounding function for taste. It looks like initiative; in review it's noise that makes diffs unreviewable and regressions hard to bisect.

The author argues for an explicit "minimal-edit" objective in prompts and harnesses: change only what the task names, leave the rest byte-identical. Worth a re-read of your own CLAUDE.md before next sprint.

--- before/handler.ts +++ after/handler.ts @@ -12,6 +12,6 @@ - const result = computeTotal(items) + const result = computeTotal(items, { rounding: 'half-even' }) -import { foo } from './foo' +import { foo, bar, baz } from './foo' // ← over-edit
"The bigger the diff, the more the human reviewer trusts the model — exactly inverted from what should happen."
